# Отчет по выполненной работе
## Теория игр, вариант 41
### Атака и защита модели ИИ (Adversarial ML)

## Введение

Данная работа посвящена решению матричной игры, описывающей конфликт между red team и службой доверия и безопасности платформы потокового видео. В терминах теории игр это двухлицевая игра с нулевой суммой: выигрыш одного игрока равен проигрышу другого, поэтому достаточно анализировать матрицу выигрышей одного игрока. В нашей постановке игрок A максимизирует число нежелательных роликов, которые успешно проходят автоматическую проверку, а игрок B стремится минимизировать этот показатель.

Для матричных игр используются следующие базовые понятия:

1. Стратегия — это правило выбора действия.
2. Чистая стратегия означает фиксированный выбор одной альтернативы.
3. Смешанная стратегия задает вероятности выбора чистых стратегий.
4. Если у игры есть седловая точка, то существует решение в чистых стратегиях.
5. Если седловой точки нет, то оптимум ищется в смешанных стратегиях.

Ключевые определения:

1. Нижняя цена игры, или maximin:
   alpha = max_i min_j a_ij

   Это гарантированный выигрыш игрока A при наихудших действиях противника.

2. Верхняя цена игры, или minimax:
   beta = min_j max_i a_ij

   Это наименьший возможный проигрыш, который может обеспечить себе игрок B.

3. Седловая точка существует, если:
   alpha = beta

   В этом случае оптимальны чистые стратегии, а значение игры равно этой общей величине.

4. Если:
   alpha < beta

   то седловой точки нет, и решение ищется в смешанных стратегиях.

5. Доминирование стратегий:
   стратегия называется доминируемой, если существует другая стратегия, которая не хуже по всем исходам и строго лучше хотя бы в одном. Доминируемые строки или столбцы можно удалить без изменения решения игры.

6. Сведение к задаче линейного программирования:
   для игрока A задача формулируется как максимизация цены игры v при ограничениях, что ожидаемый выигрыш против каждого столбца не меньше v.

7. Метод Брауна–Робинсона:
   это итерационный алгоритм, в котором игроки последовательно выбирают лучшие ответы на эмпирические частоты ходов соперника. С ростом числа итераций он приближает оптимальные смешанные стратегии и цену игры.

## Постановка задачи

По условию варианта 41 платформа потокового видео использует ML-модель для автоматической классификации и тегирования загружаемого контента.

Игрок A — red team — выбирает одну из двух атак:
1. Инъекция adversarial-примеров в видеокадры с целью обойти фильтрацию запрещенного контента.
2. Отравление обучающей выборки через массовую загрузку намеренно неверно размеченных материалов.

Игрок B — служба доверия и безопасности — выбирает одно защитное решение:
1. Adversarial training классификатора.
2. Строгая санитизация и нормализация входного видеопотока.
3. Ансамблевая мультимодельная детекция с несколькими независимыми классификаторами.

Выигрыш игрока A — число роликов с нежелательным контентом, успешно прошедших автоматическую проверку за тестовые сутки.

## Вербальная постановка → матрица

На основании условия строим матрицу выигрышей игрока A:

A =
|                | B1: adversarial training | B2: санитизация и нормализация | B3: ансамблевая детекция |
|----------------|--------------------------:|-------------------------------:|--------------------------:|
| A1: инъекция adversarial-примеров | 9  | 25 | 5  |
| A2: отравление обучающей выборки   | 21 | 5  | 17 |

Интерпретация элементов матрицы:

- 9 — при атаке adversarial-примерами и защите adversarial training проходит 9 роликов.
- 25 — при атаке adversarial-примерами и санитизации проходит 25 роликов.
- 5 — при атаке adversarial-примерами и ансамблевой детекции проходит 5 роликов.
- 21 — при отравлении выборки и adversarial training проходит 21 ролик.
- 5 — при отравлении выборки и санитизации проходит 5 роликов.
- 17 — при отравлении выборки и ансамблевой детекции проходит 17 роликов.

Таким образом, строки матрицы соответствуют стратегиям игрока A, а столбцы — стратегиям игрока B.

## Анализ доминирования

Проверим стратегии игрока A.

Первая строка: 9, 25, 5.
Вторая строка: 21, 5, 17.

Ни одна строка не доминирует другую:
- первая лучше во втором столбце;
- вторая лучше в первом и третьем столбцах.

Следовательно, доминируемых строк нет.

Проверим стратегии игрока B.

Столбцы матрицы:
- B1 = (9, 21)
- B2 = (25, 5)
- B3 = (5, 17)

Столюец В3 доминирует над В1, тк все значения в нём меньше (а следовательно лучше для игрока В)

Вывод: Столбец В1 можно не учитывать.

## Проверка седловой точки

Сначала находим минимумы по строкам:

- для первой строки min(9, 25, 5) = 5
- для второй строки min(21, 5, 17) = 5

Тогда нижняя цена игры:
alpha = max(5, 5) = 5

Теперь находим максимумы по столбцам:

- для первого столбца max(9, 21) = 21
- для второго столбца max(25, 5) = 25
- для третьего столбца max(5, 17) = 17

Тогда верхняя цена игры:
beta = min(21, 25, 17) = 17

Поскольку alpha < beta, седловой точки нет. Следовательно, чистого решения не существует, и необходимо искать решение в смешанных стратегиях.

## Графоаналитический метод

Пусть игрок A выбирает стратегию A1 с вероятностью p, а стратегию A2 с вероятностью 1 - p.

Тогда ожидаемый выигрыш против каждого столбца равен:

- f1(p) = 9p + 21(1 - p) = 21 - 12p
- f2(p) = 25p + 5(1 - p) = 5 + 20p
- f3(p) = 5p + 17(1 - p) = 17 - 12p

Игрок A стремится максимизировать наименьшее из этих значений:
F(p) = min(f1(p), f2(p), f3(p))

Оптимум находится в точке пересечения тех прямых, которые образуют нижнюю огибающую.

Решим уравнение пересечения f2(p) и f3(p):

5 + 20p = 17 - 12p
32p = 12
p = 12 / 32 = 3 / 8 = 0.375

Подставим найденное значение:

- f2(3/8) = 5 + 20 * 3/8 = 12.5
- f3(3/8) = 17 - 12 * 3/8 = 12.5
- f1(3/8) = 21 - 12 * 3/8 = 16.5

Следовательно, минимальное значение при p = 3/8 равно 12.5, и именно оно является ценой игры.

Оптимальная смешанная стратегия игрока A:
P* = (3/8, 5/8) = (0.375, 0.625)

Цена игры:
v = 12.5

Для игрока B активными оказываются стратегии B2 и B3, а стратегия B1 не используется.

Обозначим вероятности как q1, q2, q3. Тогда:
- q1 = 0
- q2 + q3 = 1

Из условия безразличия игрока A получаем:

25q2 + 5q3 = 5q2 + 17q3

Подставим q3 = 1 - q2:

25q2 + 5(1 - q2) = 5q2 + 17(1 - q2)
20q2 + 5 = 17 - 12q2
32q2 = 12
q2 = 3 / 8
q3 = 5 / 8

Следовательно:
Q* = (0, 3/8, 5/8) = (0, 0.375, 0.625)

## Сведение к линейному программированию

Для игрока A задачу можно записать как задачу линейного программирования.

Пусть p1 и p2 — вероятности выбора строк A1 и A2. Тогда необходимо максимизировать цену игры v при ограничениях:

- 9p1 + 21p2 >= v
- 25p1 + 5p2 >= v
- 5p1 + 17p2 >= v
- p1 + p2 = 1
- p1 >= 0
- p2 >= 0

После сведения к стандартной форме и решения симплекс-методом получается тот же результат:

- p1 = 0.375
- p2 = 0.625
- v = 12.5

Таким образом, решение, полученное с помощью линейного программирования, совпадает с графоаналитическим.

## Метод Брауна–Робинсона

Метод Брауна–Робинсона представляет собой итерационный способ приближения оптимальных стратегий в матричных играх. На каждом шаге один игрок выбирает лучший ответ на частотное распределение ходов соперника, а затем второй игрок делает то же самое. По мере увеличения числа итераций оценки стратегий стабилизируются и приближаются к оптимальному равновесию.

Для данной игры метод Брауна–Робинсона также приводит к следующим значениям:

- P* примерно равно (0.375, 0.625)
- Q* примерно равно (0, 0.375, 0.625)
- v примерно равно 12.5

Это подтверждает правильность результатов, полученных двумя независимыми аналитическими способами.
